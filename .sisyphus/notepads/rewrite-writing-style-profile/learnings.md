## 关键约束（从 Prometheus 计划阶段继承）

- 博客总数：184 篇（127 篇 .md + 57 篇 .markdown，排除 verify.md）

## 注意事项

- ls dir/ | wc -l 在文件名含长中文时可能因终端换行导致行数虚高，用 ls dir/*.ext | wc -l 更可靠
- 特征分析只取 ≥3 批次中反复出现的共性，防止 2008 年文章偏倚
- 原文摘录：轻量引用关键句，不大段复制
- 招牌动作、段落模板不设数量限制，有多少写多少

## [2026-03-01] Task 3 (批次B): 关键发现

- 批次B跨2008-2026存在稳定共性：动作词驱动（做/看/写/讲/用）+ 口语连接词推进（不过再想一下、话又说回来）。
- 中英混写是常态：英文术语承担技术精度（context window / retry / snapshot），中文负责论证推进与态度表达。
- 高辨识度结构为“类比起手 → 设问推进 → 回扣标题”，典型载体包括“菜刀开啤酒”“门框夹核桃”“膨胀系数”。
- 互动风格稳定：常用“我们/你/同志们”拉近距离，并用短句断言（如“并没有！”）制造节奏锚点。

## [2026-03-01] Task 2 (批次A): 关键发现

- 批次A 29篇已全部通读并落地证据文件：`.sisyphus/evidence/task-2-batch-a.md`。
- 作者稳定风格是“工程化叙述 + 口语化引导”：常用“先看/然后/最后”驱动读者跟随推理。
- 中英混用是功能性策略，不是装饰：英文用于术语对齐（如 TTL/RAG/DIP），中文用于解释与落地。
- 高识别度动作：先抛问题再自答、先下结论再用代码/反编译验证、结尾给一句可复述锤点。
- 类比来源域以日常与工程场景为主（多米诺、战场通信、队列排队），用于把抽象机制具象化。

## [2026-03-01] Task 4 (批次C): 关键发现

- 批次C（31篇）跨 2008-2025 的共性稳定：都在做“定义问题→解释机制→给出可执行下一步”。
- 词汇层长期特征：行动动词（写、跑、试、实现、解决、支持）+口语连接词（不过、然后、这样、其实、所以、最后）。
- 句式层长期特征：设问-自答高频；长短句交替；问号/叹号/省略号共同强化口语讲解感。
- 修辞层高频来源域：工程（轮子）、历史社会（风水轮流转）、日常生活（牛头卖猪肉）三类并存。
- 结构层高频模式：技术文偏“先定义问题再验收标准”，议题文偏“先源起再收束结论/留白”。
- 互动层高频模式：`我们/你/您`混用；常见自嘲或轻幽默句来降低高密度技术内容的阅读门槛。
- 招牌动作可复用配方：显式问题定义、连续失败迭代叙事、代码块后立即“翻译成人话”、结尾给行动项。

## [2026-03-01] Task 6 (批次E): 关键发现

- 批次E在跨时段混合（2008-2023）下仍保持稳定风格：问题驱动（设问）→ 机制拆解（分点/步骤）→ 落地句（结论或下一步）。
- 词汇层长期特征清晰：动作动词高频（实现/调用/传递/检查/返回），口语连接词高频（不过再想一下、那接下来、最后）。
- 中英混写是功能分工，不是装饰：英文术语负责精确定义（pending/resolved/rejected, JVM, heap usage），中文负责解释与态度表达。
- 互动与节奏锚点稳定：直接称呼读者（您/朋友/同学）+ 轻自嘲幽默（标题党、来举一个🌰）降低技术内容阅读门槛。

## [2026-03-01] Task 7 (批次F): 关键发现

- 批次F（31篇）跨 2008-2025 的稳定共性是“问题驱动 + 执行动词推进 + 结尾行动口”，体裁虽杂但推进机制一致。
- 词汇层呈现明显分工：英文术语承载精度（dynamic / bytecode / checkcast / reified），中文负责解释、态度与节奏控制。
- 句式层高频模式为“设问→拆解→回答”；长句解释机制，短句落锤定调（如“尴尬。”）。
- 修辞层高频来源域：工程建造、日常生活、社会历史并行；类比不是装饰而是降低抽象负担的主工具。
- 互动层稳定动作：你/我们/大家的称呼切换 + 自嘲幽默缓冲高密度技术表达。
- 招牌动作可复用配方：先抛机制问题、分层拆解、边界澄清（“不是…不是…”）、最后给下一步动作建议。

## [2026-03-01] Task 5 (批次D): 关键发现

- 批次D跨 2008-2026 的共性非常稳定：先给具体场景（痛点）再抽象机制，最后收敛到方法或开放提问。
- 词汇层核心是“做事型动词 + 口语连接词”：做/改/试/写/验证 与 “不过再想一下/好吧/最后”反复出现。
- 中英混写持续存在且有明确分工：英文用于术语与参数（fire and forget / viewport / review / NONBLOCKING），中文用于解释与态度。
- 句式层是“长句解释 + 短句落锤”组合，短句锤点（如“惨。”）用于强化节奏与立场。
- 修辞层高频类比来源域稳定在战争、日常工具、工作现场，典型如“战场炮火”“指甲刀刮胡子”“带实习生”。
- 招牌动作最稳定的是“二段式反转”——先肯定当前方案，再用“不过再想一下”切入更优表达或实现路径。

## [2026-03-01] Task 8 (汇总): 提炼出的核心共性列表

- 覆盖率校验通过：6 份批次报告合计覆盖 `184/184` 篇，遗漏 `0`。
- 词汇共性：动作导向动词高频（做/看/写/试/实现/验证）+ 口语连接词推进（然后/不过/那接下来/最后）+ 中英分工混写（英文术语精度，中文解释落地）。
- 句式共性：设问-自答稳定高频；长句讲机制、短句打锚点；标点口语化（问号/括号/省略号/破折号）增强现场讲解感。
- 修辞共性：类比来源域稳定覆盖工程/战争/日常；反问用于推进推理；排比用于压缩复杂度和强化结论。
- 结构共性：高频主链路为“定义问题 → 验证/拆解 → 结论/下一步”，开头常走设题/场景/立场三型，结尾偏行动项或开放问句。
- 互动共性：读者定位为同侪（我们/你/大家），以设问导航降低认知跳跃，并通过自嘲/轻幽默降低阅读压迫感。
- 招牌动作：
  - 先抛问题或立场，再用代码/实验/反编译倒推证明；
  - 把抽象机制翻译成生活画面；
  - “不过再想一下”式二次转向（B/D/E）；
  - 关键段后用短句锚点收束（如“并没有！”“尴尬。”）。

## [2026-03-01] Task 11 (profile 可用性验证): 关键发现

- 使用 `.memory-bank/writing-style-profile.md` 从头生成技术段落时，最稳的骨架是：问题句起手 → 验证路径（命令/数据）→ `不过再想一下` 二次转向 → 短句落锤。
- 与随机原文样本对比，A/C/D/G 动作最容易稳定复现；E（失败链叙事）和“我们/你/大家”互动密度是主要偏差点。
- 结论判定可采用 PARTIAL PASS：结构与语气接近作者，但需补失败链与同侪称呼频次。
